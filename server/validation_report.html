How these tests work — no real person required

The full production pipeline looks like this:

Camera → MediaPipe → landmarks / blendshapes → Classifier → label

These tests skip the Camera and MediaPipe steps entirely and inject hand-crafted numbers directly at the classifier stage. MediaPipe is Google’s library — we trust it works. What we’re testing is: given the numbers MediaPipe would produce, does our logic classify them correctly?

ClassifierWhat the test injects instead of a real person
Expression A hand-crafted Python dict of blendshape weights, e.g. {"browDownLeft": 0.9, "mouthSmileLeft": 0.3}. MediaPipe normally produces this dict from a camera frame — we just write it directly.
Blink / Gaze A synthetic 478×3 numpy array of landmark coordinates. Only the handful of indices that matter (eye corners, iris centre) are set to mathematically exact positions; everything else is zero. This gives us a known EAR value or gaze ratio without needing a face.
Posture A PostureData(shoulder_angle=20.0, head_tilt=5.0, is_slouching=True) dataclass passed straight to classify(). No pose estimation runs.
Speech tone Numpy sine waves and noise arrays (np.sin, np.random) at known amplitudes, so the RMS and zero-crossing rate are predictable.
LLM state tracker Raw JSON strings passed directly to the response-parsing function. No API call is made.
State tracker Lists of ClassifierResult objects passed to the weighted-vote function directly.

Column guide:

  • Inputs — what was passed to the classifier
  • Expected — the label it should return for those inputs
  • Actual — what it actually returned at run-time
  • Confidence — the classifier’s own score (— means N/A)
  • Status — PASS if Actual == Expected, FAIL otherwise
  • Note — why those inputs should produce that label
